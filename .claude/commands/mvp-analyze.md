Ты получил задачу для анализа: $ARGUMENTS

Выполни следующие шаги:

## 1. Изучи контекст проекта

Прочитай CLAUDE.md и ключевые файлы проекта (index.html, index.js, index.css, main.php) чтобы понять текущее состояние кодовой базы.

## 2. Запусти агента-архитектора

Используй агента `architect` чтобы проанализировать задачу и создать технический план. Передай ему:
- Описание задачи
- Контекст проекта

## 3. Сохрани результат

Создай файл `docs/mvp/<название-задачи>.md` где `<название-задачи>` — краткое имя задачи в kebab-case (например `docs/mvp/add-feedback-form.md`).

Файл должен содержать:

```markdown
# MVP: <Название задачи>

**Дата:** <сегодняшняя дата>
**Статус:** Готов к реализации

## Задача
<описание задачи как её сформулировал пользователь>

## Архитектурное решение
<вывод агента-архитектора: цель, затрагиваемые файлы, подход, шаги реализации, риски>

## Чеклист реализации
- [ ] Архитектор: план создан
- [ ] Кодер: реализация выполнена
- [ ] Ревьюер: код проверен
- [ ] Тестер: функциональность протестирована
```

## 4. Сообщи пользователю

Выведи:
- Путь к созданному файлу
- Краткое резюме плана (3-5 пунктов)
- Инструкцию: `Для реализации запусти: /implement-feature <путь-к-файлу>`
